Subsection 3.1 Active Calculus

¶

This page provides complementary information to the section Features of the text. Because this is a hosted version for the UTMOST project, it is important for you to be aware of a few additional features of the textbook.

  1. Reading this guide in its entirety, will be useful for you to take advantage of the various features of the textbook, and will make your participation in this project smoother. In other words: You need to read this guide. Seriously. All of it.

  2. This textbook was written using PreTeXt, which allows us to keep track of what features are being used. You need a login to use the UTMOST hosted version. Please make sure that you have your login credentials and that you and your students do all of your work logged in to the text. If you or your students open the textbook in another device, you will need to enter your credentials. You will see a small key on the right upper corner that indicates you have logged in.

  3. As an UTMOST user you can highlight and annotate text. When instructors highlight, they can copy the highlighted text for another purpose (e.g., send an e-mail to students). When students highlight and annotate, the system collects meta-data (the section that is annotated and the number of annotations). The text of the annotations is private to the students. Any highlighting that is done will be visible any time that the textbook is opened.

  4. An average section of the text is designed to be completed in about 2 hours of class meeting time. Each section begins with a Preview Activity that is intended for students to complete prior to class. Each is meant to be an accessible introduction to new ideas that rely largely on knowledge that students already have. Most Preview Activities can be completed in 15-20 minutes. If students are logged in to the HTML version of the text, the Preview Activities can be completed in the HTML version and the responses reviewed by the instructor. Instructors can incentivize the completion of the Preview Activities by asking the students to put their work on their desks for review in class.

  5. Each section in the textbook has three Activities (e.g., Activity 1.3.2) that are designed for being done in class by students. There is a PDF Activities Workbook that can be printed so students can use it to work on it in class.

  6. Each section of the HTML format of the textbook includes Reading Questions (just prior to the Summary). Users of the PDF format of the textbook receive a PDF document with all the reading questions of the textbook. These questions are included to encourage reading the textbook either ahead of class. The UTMOST HTML format of the textbook allows students to submit answers to these questions directly in the HTML version and have these responses saved and recorded. Once the students hit "submit answers", the instructor can immediately see the responses when logged in to their instructor account. The instructor version of the HTML textbook will show all the responses from the students who responded, each response labeled by the student ID, time, and date. Instructors might need to incentivize the completion of the reading questions by giving credit to students who complete them before class by certain time. The instructor can read the responses ahead of class and use them to tweak their plans for the day as needed. Some instructors show anonymized answers to address problems or to illustrate what an exemplary response looks like.

  7. Almost every Exercises section of the UTMOST HTML format of the textbook includes anonymous WeBWorK exercises that students can complete and receive immediate feedback with unlimited attempts without penalty.The WeBWorK exercises are intended to be more routine than the following non-WeBWorK exercises.

  8. All Exercises sections include non-WeBWorK problems that are more complex as they seek to connect multiple ideas, challenge students, and encourage them to write.

  9. Students using the HTML version of the textbook can revise the responses they provide to the Preview Activity questions before they submit them. Once they are submitted, the instructor can immediately see the responses when logged into their instructor account.

  10. This textbook includes an instructor's solution manual with answers and solutions to each Activity and non-WeBWorK exercise in the text. The answers are available in the publicly posted electronic versions (both HTML and PDF). The solutions manual is only available to instructors by direct request by email to the author (boelkinm at gvsu dot edu).

Subsection 3.2 A First Course in Linear Algebra

¶

This page provides complementary information to the section "How to Use this Book" in the Preface of the textbook. Because this is a hosted version for the UTMOST project, it is important for you to be aware of a few additional features of the textbook.

  1. Reading this guide in its entirety, will be useful for you to take advantage of the various features of the textbook, and will make your participation in this project smoother. In other words: You need to read this guide. Seriously. All of it.

If you or your students open the textbook in another device, you will need to enter your credentials. You will see a small key on the right upper corner that indicates you have logged in.

  3. This textbook does not use numbering system for the sections. The author uses acronyms to refer to each section, as they encode information about the content. The best way to handle them is to use them in class; students do not have to memorize them; only that they can recall what theorem they use. Every chapter has annotated acronyms for definitions and theorems that remind readers the importance of each of them. Although students get anxious about this, it is not such a big deal.

  4. As an UTMOST user you can highlight and annotate text. When instructors highlight, they can copy the highlighted text for another purpose (e.g., send an e-mail to students). When students highlight and annotate, the system collects meta-data (the section that is annotated and the number of annotations). The text of the annotations is private to the students. Any highlighting that is done will be visible any time that the textbook is opened.

  5. Each chapter of the HTML format of the textbook includes Reading Questions. Users of the PDF format of the textbook receive a PDF document with all the reading questions for each chapter of the textbook. These questions are included to encourage reading the textbook before class, as part of the preparation work. The UTMOST HTML format allows students to submit answers to these questions directly in the HTML version and have these responses saved and recorded. Once the students hit "submit answers", the instructor can immediately see the responses when logged in to their instructor account. The instructor version of the HTML textbook will show all the responses from the students who responded, each response labeled by the student ID, time, and date. Instructors might need to incentivize the completion of the reading questions by giving credit to students who complete them before class by certain time. The instructor can read the responses ahead of class and use them to tweak their plans for the day as needed. Some instructors show anonymized answers to address problems or to illustrate what an exemplary response looks like.

  6. The textbook has a [Reference](https://books.aimath.org/fcla/reference.html) chapter. It is organized by the major sections of the textbook. Each section contains all the symbols and terminology used in that section, with a knowl that provides its definition and an "in context" link. Each of these sections are searchable.

  7. This textbook has three categories of exercises: computational (labeled as C#), conceptual (labeled as M#), and theoretical (labeled as T#).

  8. The textbook contains a chapter on Proof Techniques that has been included to support students as they build their proving skills. Instructors can remind the students of the techniques that are available and explained in this chapter.

  9. The textbook works with examples called Archetypes, labeled with a letter, and ordered alphabetically. The Archetypes provide a range of examples that fit specific properties. They are included to serve as illustrations for how properties work or fail. They can be used to tests conjectures. The students can be encouraged to check them whenever they are learning new material. The archetypes include the relevant theorems.

  10. It is not recommended to change the order of the sections in the textbook because each chapter builds upon what has been done before.

  11. The textbook makes heavy use of Sage cells (there are 96 vignettes with Sage cells, a list of which can be found here [Sage](https://books.aimath.org/fcla/appendix-SL.html)). These are intended to help students learn to use Sage to do linear algebra and to learn how to do linear algebra by using computational examples. There is a separate GitHub repository, (Rbeezer/SLA) that contains 17 Sage worksheets that instructors can use in class (in several output formats, including Jupyter, PDF, etc.).   5. We encourage all participating instructors to incentivize the completion of the reading questions by giving some credit for submitting the responses by certain time. Users. When students are logged in to the HTML version of the text and respond to either Preview Activity questions or Reading Questions. Instructors are encouraged to use these questions with their students as well, in a similar fashion. Some instructors have used their Learning Management System to house the questions and keep track of student responses using a forum-like list, that presents all the responses without showing them to the students (or keeping them hidden until the student has submitted their own response).

  6. Be selective in what you cover in any given semester. This includes the sections of the textbook as well as the Sage exercises. You can use the chapter diagram that is presented in the Preface to make decisions about what to include or exclude.

  7. Using Sage

    The HTML format of the textbook takes advantage of the Sage programming language. Get acquainted with [how this textbook uses Sage](https://books.aimath.org/aata/sets-sage.html) to learn more about it. Instructors can either assign this section to the students, or spend some time in class to illustrate how Sage works. The cells will take Python commands.

    Each section of the HTML format of textbook contains Sage Exercises; they vary in complexity. The goal of these exercises is to get users to use technology to explore structures and make conjectures that would be very difficult to do by hand computation. For example, the Sage Exercises in Cyclic Groups were written to help readers explore the cyclic patterns of the group of units mod \(n\text{,}\) \(U(n)\text{,}\) which is sometimes cyclic, sometimes not. By using Sage, users can easily test whether or not the \(U(n)\) is cyclic for large values of \(n\) and then make a conjecture about what the general case should be.

    Sage can be used by typing a command line in a terminal window, in a Sage worksheet, and in the Jupyter notebook; however, the most friendly way to learn Sage is to use the Sage cell server. The Sage cell server allows Sage to be embedded in any web page. Users can modify and execute the Sage code. If the edited code results in disaster, one only needs to reload the web page to restore the original code. See the Sage Cell Repository for more information.
